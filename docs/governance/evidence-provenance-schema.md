# Evidence Provenance Schema

Every JSON file in `docs/verification/` and `docs/delivery/` MUST carry a top-level field:

```json
"provenance": "real" | "structural" | "dry_run" | "synthetic" | "unknown"
```

## Definitions

| Value | Meaning |
|---|---|
| `real` | Generated by an unmocked execution path against the actual subsystem |
| `structural` | Generated against an in-process fake whose response **shape** mirrors the real provider; acceptable for shape verification but NOT for release |
| `dry_run` | Generated without exercising the subsystem (placeholder run) |
| `synthetic` | Hand-injected events/state; audit only, never release-grade |
| `unknown` | Legacy artifacts without provenance field; treated as synthetic |

## Per-Gate Acceptance

| Gate | Accepted provenances | Rejected provenances |
|---|---|---|
| T3 freshness (`check_t3_freshness.py`) | `real` | `structural` (unless `t3_shape_verified` cap is explicitly accepted), `dry_run`, `synthetic`, `unknown` |
| Observability spine completeness | `real` | any other |
| Soak 24h (`check_soak_evidence.py`) | `real` (for 7×24); `dry_run` accepted as `deferred` | `synthetic`, `unknown` |
| Chaos runtime coupling | `real` with `runtime_coupled: true` | `structural`, `synthetic`, `dry_run` |
| Manifest gate | `real` | `synthetic`, `unknown` |
| Score cap | `real` | `unknown` |

## Enforcement

- `scripts/check_evidence_provenance.py` runs as a release gate.
- Artifacts without `provenance` field are moved to `docs/verification/_legacy/` during migration.
- New artifacts without `provenance` cause the gate to fail immediately.
- Score cap rule `provenance_unknown_or_synthetic` (cap 67) triggers when any gate-consumed artifact has non-`real` provenance.

## Migration Path for Legacy Artifacts

Legacy artifacts in `docs/verification/` that predate this schema should:
1. Be moved to `docs/verification/_legacy/` directory.
2. Get `"provenance": "unknown"` added if they must remain discoverable.
3. Never be used as release-grade evidence.
