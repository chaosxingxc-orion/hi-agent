## 3. Default Posture Table

Behaviour of every store/fallback/legacy path under `dev` / `research` / `prod`.

| Component | dev | research | prod | Source of truth |
|---|---|---|---|---|
| `AgentServer.__init__` durable backend missing | warn + start (`:memory:` fallback) | **raise RuntimeError** | **raise RuntimeError** | `app.py:_resolve_run_store` |
| `ArtifactRegistry` lookup of legacy tenantless artifact | visible + counter + DEBUG log | **404 + counter + WARNING log** | same as research | `registry.py:_legacy_visible` |
| `TeamRunRegistry.register` with empty `tenant_id` | **allow + DEBUG log** | **raise ValueError** | same as research | `team_run_registry.py:register` (Wave 10.2 trip-wire) |
| `SQLiteGateStore.create_gate` | spine optional, default `""` | spine optional today; advisory→blocking once orchestration callers exist | same as research | `gate_store.py:create_gate` |
| `RunQueue` enqueue without spine | not possible (`RunManager.create_run` always populates) | same | same | `run_manager.py:271+` |
| `routes_runs` POST /feedback | ✅ project_id derived from run | ✅ same | ✅ same | `routes_runs.py:344+` (Wave 10.2 fix) |
| Spine call-site governance | `python scripts/check_select_completeness.py` advisory | **blocking gate** | same | `check_select_completeness.py` (Wave 10.2 elevation) |

The governance check used to emit warnings without failing CI; downstream P0-1 explicitly required elevation. With Wave 10.2 it FAILs on any spine-missing call site that is not (a) `**kwargs`-splat (deserialization) or (b) carrying an explicit `# spine-skip: <reason>` reviewer-attested allowlist marker. False-positive surface verified by `tests/unit/test_check_select_completeness.py` (4 tests).

## 4. Server-Path Evidence

Each claim below is tagged with the verification level: **L1** = component test only; **L2** = HTTP/CLI server path; **L3** = durable end-to-end with restart survival.

### 4.1 GateStore spine round-trip — L2

- **Component**: `GateContext` (frozen) extended with `tenant_id`, `user_id`, `session_id`, `project_id` (defaults `""`); `SQLiteGateStore.create_gate` propagates kwargs onto context via `dataclasses.replace`; `_record_to_payload` includes spine in JSON; `_row_to_record` reads spine columns as source of truth (JSON used as fallback only); `get_gate`/`list_pending` SELECTs extended.
- **Tests**: `tests/integration/test_gate_store_spine.py` — 4 cases:
  - `test_gate_store_spine_roundtrip` — create → get returns full spine.
  - `test_gate_store_list_pending_returns_spine` — two tenants distinguishable in pending list.
  - `test_in_memory_gate_api_spine_propagated` — InMemoryGateAPI parity.
  - `test_gate_store_cross_tenant_denial` — HTTP 404 when tenant B requests tenant A's gate.
- **Result**: 4/4 pass (0.54 s).

### 4.2 TeamRunRegistry write — L2

- **Component**: `TeamRun` (frozen) extended with 3 spine fields (defaults `""`); `_to_row` returns a 10-tuple; `register` INSERTs spine columns and **raises `ValueError` under research/prod when `tenant_id` is empty** (Rule 6 + Rule 12); `_from_row` reads spine; `get` SELECTs spine.
- **Tests**: `tests/integration/test_team_run_registry_spine.py` — 6 cases including direct `sqlite3` row inspection, posture-aware fail-closed under research, posture-permissive under dev, cross-tenant distinct rows.
- **Result**: 6/6 pass (0.61 s); 30 pre-existing TeamRun tests unaffected.
- **Note**: zero production callers of `register()` today; the fail-closed posture branch is a trip-wire for Wave 11 PI/team orchestration when it lands, not a runtime path active under research/prod yet.

### 4.3 Feedback `project_id` from run record — L3

- **Component**: `routes_runs.py` feedback POST handler now captures the `ManagedRun` returned by `manager.get_run(...)` and reads `run.task_contract["project_id"]` (same source as `run_manager.create_run` uses to populate RunStore). The `project_id=""` + TODO is removed.
- **Tests**: `tests/integration/test_feedback_store_spine_via_http.py` — extended with:
  - `test_feedback_post_carries_project_id_from_run` — POST /runs with `project_id="proj-X"`, then POST /runs/{id}/feedback, assert stored RunFeedback row has `project_id == "proj-X"`.
  - `test_feedback_project_id_empty_when_run_unscoped` — guard against false-positive scoping when run was created unscoped.
- **Result**: 4/4 pass.

### 4.4 Spine call-site governance — L2 → blocking

- **Component**: `scripts/check_select_completeness.py` `check_spine_call_sites` elevated from advisory WARN to FAIL exit 1; adds `**word` splat skip and `# spine-skip: <reason>` allowlist marker.
- **Tests**: `tests/unit/test_check_select_completeness.py` — 4 cases including positive (failure detection), splat-skip, comment-skip.
- **Result**: 4/4 pass; running the script against current `main` produces `OK check_spine_call_sites (all spine call sites carry required kwargs)`.
- **CI**: `.github/workflows/claude-rules.yml` already wires this script in the blocking step (no `continue-on-error`).

### 4.5 Pytest verification on restricted Windows — DX

- **Component**: `pyproject.toml` pins `cache_dir = ".pytest_cache"` (in-repo); `docs/runbook/verify.md` adds PowerShell + Bash recipes using `PYTEST_DEBUG_TEMPROOT` or `--basetemp`.
- **Verification**: re-running the Wave 10.2 spine test bundle on a clean Windows checkout with restricted `%LOCALAPPDATA%` succeeds via `pytest --basetemp=.pytest_tmp ...`.

### 4.6 Wave 11 RunExecutionContext seed — L1

- **Component**: `hi_agent/context/run_execution_context.py` — frozen dataclass with `tenant_id, user_id, session_id, project_id, profile_id, run_id, parent_run_id, stage_id, capability_name, request_id`; `to_spine_kwargs()` adapter for Wave 10/10.1/10.2 store APIs; `with_stage()` / `with_capability()` ergonomic builders.
- **Tests**: `tests/unit/test_run_execution_context.py` — 5 cases.
- **Status**: declared and tested; **not yet plumbed** through routes/durable writers. Plumbing is Wave 11 work; today it gives downstream an importable target type.
