## 2. Per-Class Evidence

### Class A — Strict-posture fail-closed (P0-1 instance + 7 more)

**Root cause**: Lower layers (`build_durable_backends`, `build_artifact_registry`) raise RuntimeError/ValueError under strict posture, but callers used bare `except Exception → log.warning + None/`:memory:`` fallback, allowing the server to start with silently-degraded state.

**Instances fixed** (8 total, downstream cited 1):
1. `AgentServer.__init__` — `app.py:~1360`: re-raises under `posture.is_strict()`
2. `SystemBuilder.build_artifact_registry` — `builder.py:~381`: same pattern
3. `capability_plane_builder.py:~79`: duplicate of #2
4. `run_queue._resolve_db_path` — exception narrowed to `(ValueError, OSError)`; propagates under strict
5. `team_run_registry._resolve_team_registry_path` — same
6. `app.py` lifespan: `_feedback_store is None` under strict now raises at startup
7. `ReadinessProbe`: added `durable_backends_ok` check; strict + None backend → not-ready
8. `_check_posture` in diagnostics: now inspects live server backend state, not just env-var presence

**Tests**: `test_agent_server_research_posture_no_data_dir_fails_startup.py` (4 tests), `test_agent_server_research_posture_artifact_registry_failure_fails_startup.py` (2 tests), `test_readiness_probe_fails_when_durable_backend_missing_strict.py` (4 tests) — all pass.

### Class B — Spine completeness on all durable write paths (P0-2 instance + 8 more)

**Root cause**: 9 durable write call sites passed only business-logic fields (run_id, payload, etc.) and omitted `tenant_id, user_id, session_id, project_id` despite schemas having these columns. All durable rows had empty spine, breaking tenant isolation audit and observability.

**Instances fixed** (9 total, downstream cited 1):
1. `run_manager.py` → `RunQueue.enqueue`: all 4 fields now passed (P0-2)
2. `event_bus.py` → `StoredEvent`: contextvar `_spine_ctx` populated from auth middleware; all events get spine (system-wide fix)
3. `routes_runs.py` → `RunFeedback`: populated from `ctx` in handler
4+5. `run_finalizer.py` → `FeedbackStore.submit` + `team_space.publish`: spine from RunContext
6. `harness/executor.py` → `Artifact`: `tenant_id` from ctx
7. `team_run_registry.py`: schema migration adds `tenant_id, user_id, session_id` columns (previously absent at schema level)
8. `skill/observer.py` + `runner_telemetry.py`: `SkillObservation` gains 4 spine fields
9. `check_select_completeness.py`: extended with advisory spine-call-site check for INSERT/enqueue sites

**Tests**: `test_run_queue_spine_via_http.py` (2 tests: POST /runs → SQLite SELECT asserts spine fields), `test_feedback_store_spine_via_http.py` (2 tests) — all pass.

### Class C — Registry/Ledger interface unification (P0-3)

**Root cause**: Routes called `registry.query(..., tenant_id=ctx.tenant_id)` but `ArtifactLedger.get/query/all` had no `tenant_id` parameter — runtime TypeError 500 on production durable path.

**Fixes**:
- `hi_agent/artifacts/protocol.py` (NEW): `ArtifactStore` Protocol with consistent `tenant_id: str | None = None` signatures for all 6 methods
- `ledger.py`: `get/query/all` now accept `tenant_id`, filter accordingly; `query_by_source_ref/query_by_upstream` added
- `registry.py`: signatures aligned with Protocol
- `routes_artifacts.py`: typed against `ArtifactStore` Protocol

**Tests**: `test_artifact_routes_durable_ledger_tenant.py` (10 parametrized tests over in-memory + durable backends) — all pass, no 500s.

### Class D — Posture-aware legacy compat (P0-4 instance + 5 more)

**Root cause**: 6 read paths in `ArtifactRegistry` and `routes_artifacts._belongs_to_tenant` treated `tenant_id in ("", None, requested)` as universally visible — no posture branch. Legacy tenantless artifacts leaked across tenants in research/prod.

**Fixes** (6 sites):
- `registry.py`: `get/query/all/query_by_source_ref/query_by_upstream` — research/prod posture → deny + WARNING log + counter; dev → visible + debug log + counter
- `routes_artifacts._belongs_to_tenant` — same posture branch
- `artifacts/metrics.py` (NEW): Prometheus counters `hi_agent_legacy_tenantless_artifact_denied_total`, `hi_agent_legacy_tenantless_artifact_visible_total` (Rule 7: countable + attributable)
- `cli_commands/artifacts_migrate.py` (NEW): `hi_agent artifacts migrate-tenant --tenant-id <id> [--dry-run]`
- `docs/posture-reference.md`: legacy compat policy documented

**Tests**: `test_artifact_legacy_tenantless_posture_aware.py` (6 tests: research/dev posture behavior, counter, log assertions) — all pass.

### Class E — Delivery doc / repo HEAD drift (P0-5)

**Root cause**: `check_doc_consistency.py` could not detect (a) notice HEAD ≠ repo HEAD, (b) T3-DEFERRED paired with readiness improvement claim above 72, (c) claimed SHA not in git history.
